Executive Summary

This report provides a comprehensive strategic analysis for the Nepal STEM Alliance (NSA) to establish a formal partnership with the Ministry of Education, Science and Technology (MoEST). It covers the evolution of Nepal's CS curriculum, alignment of CodeMonkey Banana Tales with Grade 9–10 standards, NSA's collaboration models, targeted solutions to pain points, and a detailed government slide deck.

Nepal's 2026 education landscape is at a historic inflection point. Minister Pokharel's mandate calls for technology-first reform and a digital workforce prepared for the global economy. The new Grade 9–10 Computer Science curriculum (NCF 2076) has formally introduced Python programming—a transformative shift. However, a critical gap exists: the implementation layer remains largely absent.

The Solution: CodeMonkey's Banana Tales platform teaches Python through game-based, challenge-driven learning. Its coverage maps with remarkable precision to Nepal's Grade 9–10 curriculum. It is the practical implementation engine the government's curriculum reform desperately needs.

1. Old vs. New Nepal CS Curriculum

A Comparative Paradigm Shift

Nepal's computer science education has undergone its most significant structural transformation in two decades. Understanding this context is essential.

The Old Curriculum (Pre-2019)
  • Focus: MS Office tools proficiency.
  • Theoretical ICT: Rote memorization of hardware with no application.
  • No Programming: Coding absent from the curriculum.
  • Passive Learning: Students studied "about" computers.

Result: Graduates with computer literacy but zero coding ability, creating a weak STEM foundation.

The New Curriculum (NCF 2076)
  • Focus: Rigorous academic discipline.
  • Format: 128 hrs/year (64 theory + 64 practical).
  • Python Programming: 33-34 hrs of real coding.
  • Future Integration: Web Dev, AI, Data Management.

Result: Students must write, test, and debug actual Python code—not just describe it.

What the New Curriculum Demands:

Real Code

Students must write actual Python code (for, while, if/else, dictionaries).

Practical Weight

Practical work accounts for 50% of total instruction time.

Project-Based

Internal evaluations demand students build and present real programs.

3. Actionable Collaboration Models

Given the 2082/83 budget priorities, here are five concrete collaborative structures between NSA and the MoEST:

Model A

District Technology Hub Partner

NSA acts as the curriculum partner for the 100 schools receiving the NPR 2.5M grants. Costs structure ideally to the 30% software/training budget.

Model B

Teacher Bank Pedagogy Partner

Pre/in-service training for volunteer university grads to effectively teach Python instantly without prior teaching background.

Model C

National Curriculum Endorsement

Officially placing CodeMonkey Banana Tales in the MoEST list of approved/funded supplementary resources.

Model D

Book Free Friday PBL

Scaling Mayor Balen pattern nationally with a 'Coding Friday' track and 'Coding Olympics' utilizing CodeMonkey.

Model E

Pilot MoU (5-10 Govt Schools)

Immediate zero-risk pilot. NSA provides free licenses, MoEST provides access. Compare term-end learning outcomes against control schools for national scale proof.

4. Pain Point Mitigation

MoEST Problems → NSA Solutions

Stem Teacher Deficit

3,203 Tech teachers nationwide, many untrained in Python. Teacher Bank volunteers lack pedagogical setup.

➜
Zero-Coding Teacher Ready

CodeMonkey provides automated grading, lesson plans, and 3-day bootcamps making non-coding teachers facilitators instantly.

Theoretical-Only Instruction

Students read code from a book but never run it. Testing relies on rote memorization over execution.

➜
Real Execution Environment

Students must write real Python. Dashboard auto-grades real execution accuracy to feed the 50% practical score.

Remote Underutilization

Remote schools lack reliable connectivity. Hardware grants go completely unspent due to zero curriculum strategy.

➜
Turnkey Offline/Low-Bandwidth

Offline modes, pre-loaded content. Replaces idle-hardware investments with a turnkey implementation package.
